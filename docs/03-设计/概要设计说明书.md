# 概要设计说明书

**项目**：OhMyDesk — 信创内网终端远程安全管控平台
**版本**：1.0.0
**日期**：2026-06-28

## 1. 总体架构

### 1.1 系统拓扑

```
┌────────────────────────────────────────────────────────────────┐
│                        内 网 边 界                              │
│                                                                │
│  ┌──────────────────┐     WS(反连)    ┌───────────────────┐   │
│  │  Agent 客户端     │ ──────────────▶ │   Server (Relay)   │   │
│  │  Rust + Slint    │ ◀────────────── │  axum + SQLite     │   │
│  │  被控 + 主控      │  帧/键鼠/命令    │  WS中转+审计+静态  │   │
│  └──────────────────┘                └────────┬──────────┘   │
│                                               │               │
│                                    HTTP+WS    │               │
│                                  ┌────────────▼────────────┐  │
│                                  │       管理 Web           │  │
│                                  │   React + Vite          │  │
│                                  │   :8765 同源            │  │
│                                  └─────────────────────────┘  │
│                                               │               │
│                            ┌──────────────────▼──────────┐   │
│                            │     MCP Server (TS)          │   │
│                            │  5 只读工具 → Server HTTP    │   │
│                            └─────────────────────────────┘   │
│                                               │               │
│                            ┌──────────────────▼──────────┐   │
│                            │   内网 AI（Claude/私有 LLM）  │   │
│                            └─────────────────────────────┘   │
└────────────────────────────────────────────────────────────────┘
```

**设计要点**：
- **星型拓扑**：Server 为唯一中枢，Agent 主动反连（穿透内网防火墙）
- **单一内网 URL**：`:8765` 同时提供 `/`(UI) + `/api/*` + `/ws`
- **实时态在内存，历史态落库**：终端注册表 HashMap（内存）；会话/审计落 SQLite

### 1.2 模块划分

| 模块 | 仓库路径 | 职责 |
|------|---------|------|
| `protocol` | `src/protocol/` | 共享协议类型，ts-rs 导出 TS |
| `server` | `src/server/` | WS Relay + SQLite 审计 + HTTP API + 静态托管 |
| `client` | `src/client/` | Slint Agent（采集/网络/截屏/注入） |
| `admin-web` | `apps/admin-web/` | React 管理端（五页） |
| `mcp` | `apps/mcp/` | TS MCP Server（5 只读工具） |

## 2. 协议设计

### 2.1 统一信封

所有 WS 消息均采用统一 JSON 信封：

```rust
pub struct Envelope {
    pub from: String,   // 发送方 ID（"admin" 或 endpoint ID）
    pub to: String,     // 接收方 ID（"server"、"admin"、endpoint ID）
    pub ts: i64,        // 毫秒时间戳
    pub payload: Payload, // 消息体（内部 tag type 判别）
}
```

### 2.2 消息类型

| type | 方向 | 说明 |
|------|------|------|
| `register` | Agent→Server | 注册，携带 EndpointInfo |
| `register_ack` | Server→Agent | 注册确认，携带分配 ID |
| `heartbeat` | Agent→Server | 心跳保活（30s） |
| `endpoint_list` | Server→Admin | 终端列表推送 |
| `connect_request` | Admin→Server | 发起远控请求 |
| `auth_result` | Agent→Server | 目标端授权结果 |
| `connect_ack` / `reject` | Server→Admin | 连接成功/被拒绝 |
| `frame` | Agent→Admin | 画面帧（JPEG base64） |
| `input` | Admin→Agent | 键鼠输入事件 |
| `screenshot_req` | Server→Agent | 截图请求（批量/会话内） |
| `screenshot_resp` | Agent→Server→Admin | 截图回传 |
| `session_end` | 双向 | 会话结束通知 |
| `file_transfer_*` | 双向 | 文件传输系列消息 |
| `command_exec` / `command_result` | Admin↔Agent | 远程命令执行 |

## 3. 数据模型

### 3.1 SQLite 表结构（`scripts/db/schema.sql`）

```sql
-- 终端台账（补充内存注册表的持久化信息）
CREATE TABLE IF NOT EXISTS endpoints (
    id TEXT PRIMARY KEY,
    name TEXT,          -- 使用人/主机名
    ip TEXT,
    mac TEXT,
    os_name TEXT,
    os_type TEXT,       -- kylin/uos/windows/ubuntu/macos
    cpu_model TEXT,
    cpu_arch TEXT,      -- loongarch64/aarch64/x86_64
    ram_mb INTEGER,
    gpu_model TEXT,
    gpu_vram_mb INTEGER,
    last_seen INTEGER   -- epoch 秒
);

-- 会话历史
CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    mode TEXT,          -- A 或 B
    from_id TEXT,
    to_id TEXT,
    start_at INTEGER,
    end_at INTEGER,
    status TEXT         -- active/ended/rejected
);

-- 审计日志
CREATE TABLE IF NOT EXISTS audit_logs (
    id TEXT PRIMARY KEY,
    session_id TEXT,
    ts INTEGER,
    actor_id TEXT,
    event_type TEXT,    -- connect/auth_fail/reject/screenshot/input/disconnect/command/file_transfer
    text TEXT
);

-- 系统设置（管理员账号/密码/JWT 密钥）
CREATE TABLE IF NOT EXISTS settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

### 3.2 信创标识推断规则

| OS 标识 | 推断逻辑 |
|---------|---------|
| 麒麟(Kylin) | os_name 含 "Kylin" |
| 统信(UOS) | os_name 含 "UOS" 或 "Uniontech" |
| Windows | os_type = "windows" |
| Ubuntu/Linux | 其他 Linux |

| CPU 架构 | 推断逻辑 |
|---------|---------|
| 龙芯 LoongArch | arch = "loongarch64" |
| 鲲鹏/华为 AArch64 | arch = "aarch64" + cpu_model 含 "Kunpeng" |
| 飞腾 | arch = "aarch64" + cpu_model 含 "Phytium" |
| x86-64 通用 | arch = "x86_64" |

## 4. 服务端详细设计

### 4.1 模块职责（`src/server/src/`）

| 文件 | 职责 |
|------|------|
| `main.rs` | axum Router 配置；静态托管 admin/dist；启动 |
| `hub.rs` | WS 连接管理 + 信封路由 + 广播（HashMap<id, Sender>） |
| `registry.rs` | 内存终端注册表；在线超时检测（90s） |
| `session.rs` | 会话建立/鉴权（A/B 模式）/结束 |
| `audit.rs` | SQLite 文本审计落库 + 事件聚合 |
| `db.rs` | SQLitePool 连接池（sqlx） |
| `http.rs` | `/api/*` 只读 HTTP（供 MCP + 管理端查历史） |
| `auth.rs` | JWT 签发验证（HS256），`/api/*` 和 admin WS 鉴权 |

### 4.2 WS 连接鉴权

- **Agent 注册**：无需 Token，发 `register` 消息即接入
- **Admin 连接**：URL 携带 `?token=<jwt>`，服务端握手时验证，失败 close 1008
- **HTTP API**：`Authorization: Bearer <jwt>` Header

## 8. 安全设计

| 层面 | 措施 |
|------|------|
| 传输 | TLS（rustls/SChannel），wss:// |
| 认证 | JWT HS256，默认 24h 有效期；`OHMYDESK_JWT_SECRET` 环境变量 |
| 授权 | Agent 注册无需登录；远控操作 Admin Token 必须 |
| 被控保护 | 目标端弹窗授权，拒绝即记录 auth_fail 审计 |
| 数据隔离 | 纯内网部署，MCP Server 也在内网；不外传 |

## 9. 部署设计

```
内网服务器
├── ohmydesk-server    # Rust 单二进制（systemd 守护）
├── ohmydesk.db        # SQLite 数据文件（挂卷持久化）
└── admin-web/dist/    # 由 server 同端口静态托管

终端 PC（信创）
└── ohmydesk-client.deb / ohmydesk-client.exe
    └── 内嵌 wss://服务器/ws 地址，dpkg 安装即自启
```

**环境变量**：

| 变量 | 说明 | 默认 |
|------|------|------|
| `OHMYDESK_WEB_DIR` | 管理端静态目录 | `src/admin-web/dist` |
| `DATABASE_URL` | SQLite 路径 | `sqlite:./ohmydesk.db` |
| `OHMYDESK_JWT_SECRET` | JWT 签名密钥（生产必设） | 随机生成（每次重启失效） |
| `OHMYDESK_FAKE_CAPTURE` | WSL 环境合成占位帧 | 空（真实截屏） |
| `OHMYDESK_SERVER` | 客户端连接 URL | `wss://rc.guoziweb.com/ws` |
