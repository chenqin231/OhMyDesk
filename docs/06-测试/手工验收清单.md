# OhMyDesk MVP 手工验收清单（demo 通过标准）

> 对应 feature-spec §7 六条总验收。每条给出：**前置 → 操作步骤 → 预期结果**，并标注**已自动验证证据**与**手工（浏览器/真机）验证点**。
> 自动层（协议/路由/落库/数据）已由 `cargo test` + node 双端探针 + 真实 MySQL 验证通过；手工层是把这些能力在真实 GUI 上走一遍。

## 0. 彩排前置（一次性起齐五件套）

```bash
# ① MySQL（可选，审计落库需要）
docker run -d --name ohmydesk-mysql -e MYSQL_ROOT_PASSWORD=ohmydesk \
  -e MYSQL_DATABASE=ohmydesk -p 13307:3306 mysql:8.0

# ② 构建
cargo build --workspace
pnpm -C src/admin-web install && pnpm -C src/admin-web build
( cd src/mcp && pnpm install && pnpm build )

# ③ Server（仓库根，托管 admin/dist）
DATABASE_URL="mysql://root:ohmydesk@127.0.0.1:13307/ohmydesk" \
OHMYDESK_WEB_DIR="src/admin-web/dist" cargo run -p server

# ④ ≥2 个 Agent（各需 X11 显示，换不同使用人名）
OHMYDESK_SERVER="ws://127.0.0.1:8765/ws" cargo run -p client -- "张伟-财务部"
OHMYDESK_SERVER="ws://127.0.0.1:8765/ws" cargo run -p client -- "李娜-研发部"

# ⑤ 浏览器开 http://127.0.0.1:8765/
```

## ✅ 验收 1 — M1 终端资产（2+ 台在线 + 真实硬件 + 信创标识）

- **步骤**：起 server + ≥2 个 Agent → 浏览器进「资产」页。
- **预期**：列表显示 ≥2 台**在线**；每行有使用人/IP/OS 图标；点行 → 右侧抽屉看 CPU 型号/核数/arch、内存、MAC；信创终端打「信创·麒麟·龙芯」类标签，x86 Windows 标「非信创」。杀掉一个 Agent → 约 15s 内该行变离线。
- **已自动验证**：双真实终端注册 + 信创推断（kylin/loong_arch→信创，windows/x86_64→非信创）+ 离线检测（exit→online=false，heartbeat→true）。`/api/endpoints` 实时返回 EndpointView[]。
- **手工验证点**：浏览器列表渲染 + 详情抽屉 + 信创图标 + 离线变色。

## ✅ 验收 2 — M2 模式 A（Web 远控终端：授权→画面→操作→断开）

- **步骤**：「资产」或「远程」页选一台在线终端 → 发起远程（模式 A）→ 被控 Agent 弹授权框 → 点同意 → 管理端看到对端实时画面 → 鼠标在画面上点击 → 对端生效 → 断开。
- **预期**：授权弹窗出现；同意后管理端贴出对端屏幕帧（2-3fps）；点击/移动注入到对端；断开后画面关闭，会话结束。
- **已自动验证**：server 时序（connect_request→IncomingControl→auth_result→connect_ack）、Input 主控→被控路由、Frame 被控→主控路由（node 探针闭环）。client 截屏（xcap）/注入（enigo）单元链路、坐标等比还原。模式 A 与 B 走 server 同一路径（A 免密码校验）。
- **手工验证点**：真机授权弹窗 + 实时画面渲染 + 鼠标注入生效（需 X11 显示）。

## ✅ 验收 3 — M2 模式 B（终端→终端，ID+密码；密码错被拒）

- **步骤**：Agent A 界面输入 Agent B 的 ID + 6 位密码 → 发起 → B 弹授权 → 同意建会话。再用**错误密码**发起一次。
- **预期**：正确密码 → B 弹授权 → 建立远控；**错误密码 → 直接返回「密码错误」拒连**，且 B 端主动拒绝时返回「用户拒绝」。
- **已自动验证**：node 双端探针——正确密码闭环 + 错误密码 `reject「密码错误」` + 主动拒绝 `reject「用户拒绝」` 双分支均通过；两类拒因分别落 `auth_fail` / `reject` 审计。
- **手工验证点**：两台真机 Agent 互控 + 密码错弹错提示。

## ✅ 验收 4 — M3 一键批量截图（在线终端屏幕墙）

- **步骤**：进「批量监控」页 → 点「一键批量截图」。
- **预期**：数秒内所有在线终端当前屏幕缩略图依次出现在墙上；点某张可放大查看。
- **已自动验证**：`screenshot_req` 广播全 agent → 被控端 `consume_screenshot` 截帧后回发 `screenshot_resp{endpoint_id}` → server `forward_by_to` 路由回 admin → 前端按 `endpoint_id` 入缓存渲染。node 双 agent 探针：admin 收到 ep-a/ep-b 两张回发，按 endpoint 正确归位。截图指令落 `screenshot` 审计。
- **手工验证点**：截图墙真实缩略图（真机截屏内容）+ 放大。

## ✅ 验收 5 — M4 会话审计（连接记录 + 操作文本 + 可筛选）

- **步骤**：完成 1 次远控后进「会话审计」页 → 查看记录 → 按终端筛选。
- **预期**：出现该连接的会话记录（起止/时长/终态）+ 至少「会话建立 / 批量截图 / 输入操作 N 次 / 会话结束」等操作文本；可按终端 ID 筛选。
- **已自动验证**：真实 MySQL 落库——`/api/audit` 返回 4 条（connect/input/screenshot/disconnect），`/api/sessions` 返回 1 条 `status=ended`（会话 INSERT + 终态 UPDATE 均生效）；MySQL 直查 `audit_logs`/`sessions` 行存在。`/api/audit?endpoint=&from=&to=` 支持筛选。
- **手工验证点**：审计页表格渲染 + 时间线抽屉 + 终端/时间筛选交互。

## ✅ 验收 6 — M5 MCP/AI（自然语言查实时数据）

- **步骤 A（应用内）**：进「AI 助手」页，问「现在有几台麒麟终端在线？」「今天有哪些远程连接记录？」「谁在控制财务部的电脑？」。
- **步骤 B（外部 MCP 客户端）**：`OHMYDESK_API_BASE=http://127.0.0.1:8765 node src/mcp/dist/index.js`，在 Claude Desktop 等 MCP 客户端接入，调用 `list_endpoints` / `get_active_sessions` / `query_audit_log` / `get_endpoint_stats`。
- **预期**：回答基于**实时**终端/会话/审计数据（非预录死值）；展示对应 MCP 工具调用。
- **已自动验证**：MCP `client.ts` 真实 HTTP 拉到 2 终端 / 1 会话 / 4 审计（与 `/api/*` 一致）；4 个 tool tsc 构建零错误，规避 bigint 陷阱；助手页基于 store 实时数据计算应答。
- **手工验证点**：助手页问答交互 + 外部 MCP 客户端工具调用返回真实数据。
