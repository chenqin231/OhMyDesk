## T1 — ④a 服务端会话泄漏根治（TDD，server 独立）

**文件**：`src/server/src/session.rs`（加方法+测试）、`src/server/src/hub.rs`（加 async end_client_sessions+测试）、`src/server/src/main.rs:294`（断开时调用）。

- `SessionStore::remove_sessions_of(&self, client_id, now, status) -> Vec<(Session, String)>`：移除并返回所有 `from_id==client_id || to_id==client_id` 的会话（set end_at/status，附输入摘要）。
- `Hub::end_client_sessions(&self, client_id, now).await`：对每条被移除会话，向对端（≠client_id 的一方）发 `Message::SessionEnd{session_id}`，`audit.end_session` 落库 + 审计 Disconnect「对端断开」。镜像现有 `handle_session_end` 的通知+落库写法。
- `main.rs` 断开处（~294）：`remove_client` 前调用 `hub.end_client_sessions(id, now_sec()).await`。

**测试**：session.rs 单测（含 client 作 from/to 两侧 + 无关会话，只移除应移除的）；hub.rs 测试（ep-a→ep-b active，断开 ep-a → ep-b 收 SessionEnd 且会话不再 active）。

## T2 — ④b 聊天漂移最小对齐修（TDD，client + server 测试）

- 校验 set 侧（ui_glue 599/614）与 controlled（dispatch 89/116）set 点一致；不一致则一并对齐。

**测试**：client 漂移序列（控制 S1 → 重控 S2 → 迟到 SessionEnd{S1}）后 ctrl_session 必等 S2；server route 测试（被控 to_id → 主控 from_id=admin-* 的 ChatMessage 正确投递）。
