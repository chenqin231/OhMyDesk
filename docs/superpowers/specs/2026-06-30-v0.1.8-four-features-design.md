# OhMyDesk v0.1.8 四功能设计

> 状态:已确认设计,待转实现计划
> 日期:2026-06-30
> 范围:剪贴板双向同步、WEB 强制远程、客户端伙伴密码选填、临时密码 UI 改造

## 1. 概述与目标

本迭代交付 4 个相互独立但部分共享授权模型的功能:

1. **剪贴板双向同步**:远程会话内,主控 ↔ 被控自动实时同步纯文本剪贴板。
2. **WEB 强制远程**:资产列表「远程控制」改名「申请远程」(需被控同意),新增「强制远程」(免被控同意,加二次确认)。
3. **客户端伙伴密码选填**:有密码且正确 → 免对方同意直连;无密码 → 需对方同意。
4. **临时密码 UI**:默认隐藏(眼睛图标切换明文),「复制」「换」改为 Unicode 图标,功能不变。

功能 2、3 共享同一套**统一授权模型**(见 §3),是本迭代的设计核心。

## 3. 设计总则:统一授权模型

**决策位置**:server `handle_connect_request`(`handlers.rs:30-102`)一处收敛。

**`auto_accept` 真值表**:

| mode | 凭证 | `auto_accept` | server 行为 |
|---|---|---|---|
| B | `password` 存在且 `check_password` 通过 | true | 建 Active 会话 + 回主控 `ConnectAck` + 发被控 `IncomingControl{auto_accept:true}` |
| B | `password` 为 None/空 | false | 走同意流程:发被控 `IncomingControl{auto_accept:false}`(现状) |
| B | `password` 存在但校验失败 | — | 回 `Reject{reason:"密码错误"}` |
| A | `force==true` 且发起方 `admin-` 前缀 | true | 同上 auto_accept 直连 |
| A | `force==false` | false | 走同意流程 |
| A | `force==true` 但发起方非 `admin-` | — | 回 `Reject`(防越权强制) |

**关键行为变更**:
- 现状「模式 B 无密码 → `unwrap_or("")` 校验失败 → Reject」**改为**「模式 B 无密码 → 跳过密码校验 → 走同意流程」。
- 现状「密码对了仍弹框同意」**改为**「密码对了 → auto_accept 免同意」。

**auto_accept 路径的数据流**(区别于现状「等 AuthResult 才回 ConnectAck」):
1. server 在 `handle_connect_request` 内即:建 **Active** 会话 + 落 `Connect` 审计(标记 `forced`/`auto` 来源)+ 回主控 `ConnectAck` + 发被控 `IncomingControl{auto_accept:true}`。
2. 被控收 `IncomingControl{auto_accept:true}`:**不弹框**,直接进被控态(复用 `dispatch.rs:362-367` 同意路径:`CAPTURE_CTRL` Start + 设 `controlled`),并弹非阻塞提示「正在被 {from} 远程控制」(走 `ToUi::Notice`/`BeingControlled`)。
3. 主控收 `ConnectAck` → `ToUi::RemoteAck`(现状不变)。

**审计**:强制/免同意连接照常落 `Connect`;强制远程在审计中标记发起方身份,满足内网管控可追溯。

### 模块 D:剪贴板双向自动同步(功能 1)

**核心约束**:`arboard::Clipboard` 句柄非 `Send` 且 Linux/X11 下剪贴板所有权要求进程常驻持有(进程/句柄 drop 可能丢内容)。故剪贴板读写**全部收敛到一个独占常驻线程**(与 `consume_inject`/`consume_capture` 的 enigo/xcap 独占线程同构),`last_synced` 仅存活于该线程内 —— **无需任何跨线程锁,天然无回环竞态**。

- `src/protocol`:`Message::ClipboardSync`(模块 A 已含)
- `src/client/src/net/mod.rs`:`FromUi::ClipboardSync{session_id, text}`(出站)。`ToUi` **不新增**。
- `src/client/src/workers.rs`:新增 `consume_clipboard`,内部起独占线程持有 `arboard::Clipboard` + `last_synced: String`:
  - **读路径(poll)**:`loop{ sleep(~500ms); }` —— 会话激活时读本地剪贴板 text,`text != last_synced && !text.is_empty()` → `from_ui_tx.send(FromUi::ClipboardSync{session_id, text})` + `last_synced = text`
  - **写路径(收对端)**:经新 bridge `CLIPBOARD_TX`(`OnceLock<std::sync::mpsc::Sender<String>>`,仿 `INJECT_TX`)从 dispatch 收到对端文本 → `clipboard.set_text(text)` + `last_synced = text`(关键防回环:写本地后下次 poll 读到 == `last_synced`,不回发)
  - 读 poll 与写 bridge 在**同一线程**内交替处理(mpsc `try_recv` 非阻塞 + sleep 节奏),保证 `last_synced` 单线程独占
  - 会话激活 + session_id 来源:worker 持 `active: Arc<Mutex<Option<String>>>`,由会话建立处(被控 auto_accept/同意进被控态、主控收 ConnectAck→RemoteAck)写入、会话结束处清空并重置 `last_synced=""`。被控态可直接复用 `CAPTURE_CTRL` Start/Stop 时机同步更新;主控态在 `RemoteAck`/`SessionEnded` 处更新。
  - Wayland:`is_wayland_session()` 特判退化(读写都跳过),每会话告警一次(`notified_for` 模式)
  - 超大文本(>256KB)跳过,记一次 debug(防刷帧)
- `src/client/src/net/dispatch.rs`:
  - 出站映射:`FromUi::ClipboardSync` → `Envelope{to:None, Message::ClipboardSync{...}}`(仿 Frame `dispatch.rs:389-406`)
  - 下行处理:收 `Message::ClipboardSync{session_id, text}` → 校验方向(`controlling==session_id || controlled==session_id`)→ `CLIPBOARD_TX.with_send(text)`(交给 worker 线程写本地 + 更新 `last_synced`,**不在 dispatch 直接碰 arboard**)
- `src/server/src/hub.rs`:route_to_peer 加臂(模块 B 已含)
- `src/client/src/main.rs`:`rt.spawn(workers::consume_clipboard(...))` 挂载;`CLIPBOARD_TX` 在 worker 启动时 set(仿 `INJECT_TX`)

> 防回环本质:`last_synced` 单线程私有,收(写本地)与发(poll)都更新它,值相同即不发 —— 不存在跨线程读写竞态,也不存在 A→B→A 回弹。

## 6. 错误处理与边界

- **强制远程越权**:模式 A force 但发起方非 `admin-` → server Reject,前端按钮仅 admin 可见已有约束,server 端再硬校验(纵深防御)。
- **被控离线**:申请/强制远程对离线终端按钮 disabled(现状 `status==="offline"`)。
- **剪贴板回环**:`last_synced` 收发共用实例双向更新(§4 模块 D)。
- **剪贴板 Wayland 不可用**:特判退化,不阻断会话,每会话告警一次。
- **剪贴板大文本/非文本**:仅同步纯文本;超大文本(如 >256KB)截断或跳过(防刷帧),记一次 debug。
- **会话结束**:清 `last_synced`,停止剪贴板同步(随 controlling/controlled 清空)。
- **旧客户端兼容**:`force` 缺省 false;旧被控端不识别 `auto_accept` 字段时 serde 反序列化失败风险 —— 故 `IncomingControl.auto_accept` 也应 `#[serde(default)]`,旧被控端落到 false(走弹框,行为安全降级)。

- **手动**:跨平台剪贴板(Linux X11 ↔ Windows);临时密码掩码/眼睛切换;强制远程二次确认

## 8. 实现分组与顺序

| 组 | 内容 | 依赖 |
|---|---|---|
| 1 | 模块 A 协议 + ts-rs 导出 | 无(其余都依赖) |
| 2 | 模块 B+C+F+G 授权模型(功能 2+3,server+client+admin-web) | 组 1 |

建议 worktree 隔离,分组提交;组 2 是授权安全核心,需重点测试。

## 9. 风险与缓解

- **授权语义变更**(密码连免同意):正是需求;审计保留 `Connect`,强制标记发起方,可追溯。
- **强制远程滥用**:仅 admin- 可用 + server 硬校验 + 前端二次确认。
- **剪贴板回环/隐私**:`last_synced` 单线程私有缓存;仅会话期同步;纯文本;超大跳过。
- **协议兼容**:`force`、`auto_accept` 均 `#[serde(default)]`,旧端安全降级到「需同意」。
- **Slint 图标渲染**:emoji 不可用,优先已验证几何字符,真机验证缺字形则退化短文字(见模块 E)。
