# OhMyDesk v0.1.8 交互体验设计

> 设计师产出(Dieter Rams 十原则 + Nielsen 启发式)。配套:`2026-06-30-v0.1.8-four-features-design.md`
> 日期:2026-06-30 | 范围:授权模型变更引入的界面交互态

## 1. 设计原则与范围

授权模型从「全部需同意」分裂为**免同意(密码对/强制)**与**需同意(无密码/申请)**两条路径,但现有 UI 仍是单一遮罩 + 单句文案,且无取消/超时、被控免同意时零感知。

**原则**:
- **诚实**(Rams #6):被控端任何情况下都必须知道「自己正在/即将被谁控制」。
- **系统状态可见**(Nielsen #1):主控「请求已送达、在等人」与「卡住」必须可区分。
- **少即是多**(Rams #10):不堆新弹窗,让现有遮罩/弹窗/横幅按路径自适应。

**已确认的 4 项交互决策**:
1. 无密码申请等待对方同意:**60s 超时**自动取消。
2. 被控授权弹窗:**60s 倒计时**自动拒绝。
3. 被控免同意被控:**常驻顶部横幅**告知,强制时红点 + 标记。
4. 复制 ID/密码:按钮短暂变 **「已复制 ✓」**(0.8s)。

## 2. 主控发起状态机

```
                       点击「开始控制」
                            │
              ┌─────────────┴─────────────┐
         填了密码                       未填密码
       (预期免同意)                   (申请,需同意)
            │                              │
   遮罩②「建立加密连接…」          遮罩①「已发送请求,等待同意」
            │                         + 60s 倒计时 + [取消请求]
            │                              │
     ┌──────┼──────┐            ┌──────────┼──────────┬─────────┐
  ConnectAck Reject          ConnectAck  Reject   倒计时=0    点取消
  (密码对)  (密码错)         (对方同意) (对方拒绝) (超时)    (主动)
     │        │                  │          │         │         │
  进画面   遮罩③               进画面    遮罩③      遮罩③    复位空闲
        「密码错误」                     「对方拒绝」「对方未响应」 + 收尾会话
```

**关键约束(协议层)**:无密码申请时,主控在对方同意前**收不到 session_id**(server 生成、随 ConnectAck 才下发)。因此:
- 「取消请求」/「超时」时主控**无 session_id 可发 SessionEnd** → 只本地复位 + 置「已取消」标记。
- 若对方在主控取消后才同意,server 仍回 ConnectAck(带 session_id)→ 主控收到时若「已取消」,**不进主控态,反而用该 session_id 发 SessionEnd 收尾 server 会话**(防止「幽灵会话」+ 远程视图复活,复用既有 ended_session 丢帧机制的思路)。
- 被控端弹窗 60s 自动拒绝是 server 会话的兜底收尾(与主控 60s 对齐)。

## 3. 界面态线框 + 落地点

### ① 主控 · 无密码申请等待(新)
```
┌───────────────────────────────┐
│            ◷                   │   脉冲点(非 ↻,区别于「连接中」)
│   已向 王工@财务 发送控制请求      │
│       等待对方在终端上点击同意      │
│   对方未操作?可电话告知对方        │
│        ⏱ 27s   [ 取消请求 ]      │
└───────────────────────────────┘
```
落地:`app.slint` 遮罩 ③(`:768-822`)内按 `awaiting_consent` 切换文案 + 显示 `consent_countdown` + 取消按钮。Slint `Timer{ interval:1s; running: root.awaiting_consent; }` 每秒递减;到 0 触发取消回调。

### ② 主控 · 有密码免同意直连
```
┌───────────────────────────────┐
│            ↻                   │
│      正在建立加密连接…           │   不提「等待授权」
└───────────────────────────────┘
```
落地:`connecting && !awaiting_consent` → 现有遮罩,文案改为「正在建立加密连接…」(去掉「等待对端授权」)。

### ③ 主控 · 终态提示(复用 remote_status)
| 场景 | 文案 |
|---|---|
| 密码错 | ✕ 密码错误 |
| 被拒 | ✕ 对方拒绝了控制请求 |
| 超时 | ⧖ 对方未响应(60s 未同意),请求已取消 |

落地:`remote_status` 文本(`app.slint:521-524` danger Text),由 net 侧 `ToUi::RemoteRejected{reason}` 驱动。

### ④ 被控 · 授权弹窗增强(来源 + 倒计时)
```
┌─────────────────────────┐
│           ?              │
│      远程控制授权请求       │
│  王工@财务 请求远程控制本机 │
│  来源:终端伙伴 / 管理员    │
│   ⏱ 53s 后自动拒绝         │
│   [  拒绝  ] [   同意   ]  │
└─────────────────────────┘
```
落地:`app.slint` 授权弹窗(`:824-906`)加来源行 + 倒计时行。新增属性 `auth_source: string`、`auth_countdown: int`;`Timer{ interval:1s; running: root.auth_pending; }` 递减,到 0 触发 `auth_reject()`。来源由 `ToUi::ControlRequest` 带 `source` 字段(mode A→「管理员」,mode B→「终端伙伴」)。

### ⑤ 被控 · 免同意横幅(强制/密码,新)
```
普通免同意(密码): ● 此设备正在被 王工@财务 远程控制      [我要断开]   (琥珀)
管理员强制:        ● 管理员正在强制远程控制本机           [我要断开]   (红)
```
落地:`app.slint` 被控横幅(`:347-369`)增强:
- 新增属性 `controlled_forced: bool`;`forced` → 背景/点/文字用 `Theme.danger` 系 + 文案「管理员正在强制远程控制本机」;否则现状琥珀。
- 加「我要断开」按钮 → 新回调 `stop_being_controlled()`,被控端发 `SessionEnd`(用 `ctrl_session` 持有的 controlled session_id)。
- 数据来源:`ToUi::BeingControlled` 加 `forced: bool`(由 IncomingControl 的 `auto_accept && mode==A` 推断;mode B 密码免同意 forced=false)。

### ⑥ 临时密码行(功能 4 交互细节)
- 默认 `••••••`;眼睛 `◉`/`○` 切换明文(spec 模块 E 已定,真机验证字符)。
- 复制:点击后按钮 label 0.8s 变 `✓`(或「已复制」)再回弹。落地:复制按钮局部 `property <bool> copied` + `Timer{ interval:0.8s; running: copied; triggered => { copied=false; } }`;label `copied ? "✓" : "⧉"`。ID 复制按钮同此模式。

## 4. 协议 / 状态影响(增量,补充到主 spec/plan)

新增/变更(均向后兼容,`#[serde(default)]` 或仅客户端内部 ToUi):
- `ToUi::ControlRequest` 加 `source: String`(「管理员」/「终端伙伴」)——仅客户端内部枚举,不过线。
- `ToUi::BeingControlled` 加 `forced: bool`——仅客户端内部。
- `FromUi` 加 `CancelRemote`(主控取消申请)、`StopControlled{session_id}`(被控主动断开)。
- net 连接级维护「申请已取消」标记:收到迟到 ConnectAck 时据此发 SessionEnd 收尾、不进主控态。
- `IncomingControl.mode` + `auto_accept` 已能推导 source/forced,无需新增过线字段。

> 主控「填没填密码」UI 已知(`target_password` 是否空),`connect_b` 时据此置 `awaiting_consent`,**无需新增过线协议**。超时/倒计时纯客户端 Timer。

## 5. 动效与节制(软渲染约束)

- Slint `renderer-software`,**不做无限旋转动画**(现状 ↻ 为静态字符)。倒计时用每秒整数递减(Timer 1s),非平滑动画。
- 脉冲点 ◷:静态字符即可,不强求呼吸动效。
- 复制 ✓ 反馈:状态切换(无过渡动画),0.8s 后回弹。

## 6. 决策记录

| # | 决策 | 选择 | 依据 |
|---|---|---|---|
| 1 | 申请超时 | 60s 自动取消 | 给对方足够反应时间,避免永久转圈 |
| 2 | 弹窗倒计时 | 60s 自动拒绝 | 与主控对齐;无人值守默认安全(拒绝) |
| 3 | 被控知情 | 常驻横幅 + 强制红标 | Rams 诚实;随时可断开 |
| 4 | 复制反馈 | 按钮变 ✓ 0.8s | 最小反馈,无新组件 |
