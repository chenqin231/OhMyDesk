# 需求规格说明书

**项目**：OhMyDesk — 信创内网终端远程安全管控平台
**版本**：1.0.0
**日期**：2026-06-28
**状态**：已锁定（参赛版本）

---

## 1. 背景与范围

### 1.1 背景

企业内网（物理隔离）场景下，员工终端 PC 的远程协助和安全管控长期处于工具空白：消费级远控依赖公网、堡垒机管不到 PC 终端。信创国产化（党政 2026 年 80% 目标）进一步放大了麒麟/统信 OS + 龙芯/鲲鹏 CPU 架构下的适配缺口。

### 1.2 范围

本说明书覆盖 OhMyDesk 参赛版本（demo 功能），不含：录像回放、真实 NAT 穿透、多显示器、批量脚本下发、违规外联检测。

---

## 2. 用户角色

| 角色 | 描述 | 主要操作 |
|------|------|---------|
| **平台管理员** | IT 管理人员，使用 Web 管理端 | 查看终端、发起远控、批量截图、查审计、问 AI |
| **终端用户** | 员工，其 PC 被 Agent 纳管 | 接受/拒绝远控请求，或作为主控端发起 B 模式远控 |

## 3. 功能需求

### 3.1 M1 终端资产管理

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| M1-1 | Agent 启动后自动反连服务端，完成 WS 注册，上报：IP、MAC、主机名/使用人、OS 版本、CPU 型号/架构、内存大小、GPU 型号/显存 | P0 |
| M1-2 | 服务端维护内存终端注册表，周期心跳（30s）超时（90s）判断离线 | P0 |
| M1-3 | 管理端展示终端列表：设备 ID、使用人、IP、OS、CPU、在线状态（绿/灰） | P0 |
| M1-4 | 信创标识自动识别：OS 类型（麒麟/统信/Windows/Ubuntu/macOS）、CPU 架构（龙芯LoongArch/鲲鹏AArch64/飞腾/x86-64）图标显示 | P0 |
| M1-5 | 点击终端行展开侧边抽屉，显示完整硬件画像 | P1 |

### 3.2 M2 远程控制

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| M2-1 | **模式 A**：Web 管理员点"远程"→目标 Agent 截屏→base64→WS→Web 渲染；鼠标坐标/键盘事件回传 Agent 注入 | P0 |
| M2-2 | **模式 B**：客户端 A 输入目标设备 ID + 临时密码→服务端鉴权路由→建立会话，画面流同模式 A | P0 |
| M2-3 | 连接前目标端弹授权对话框（接受/拒绝）；拒绝则连接失败并记录 | P0 |
| M2-4 | 会话存续期间：发起方持续收帧（screenshot_resp），鼠标移动/点击/键盘事件经 WS 回传目标执行 | P0 |
| M2-5 | 任一端断开触发 session_end，双端通知，会话落库终态 | P0 |

### 3.3 M3 批量监控截图

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| M3-1 | 管理端"批量截图"按钮：向所有在线 Agent 广播 screenshot_req | P0 |
| M3-2 | 各 Agent 截当前屏→JPEG base64→WS 回传服务端→推送管理端 | P0 |
| M3-3 | 管理端以网格墙展示所有终端截图，每格显示终端名称/时间戳 | P0 |

### 3.4 M4 会话审计

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| M4-1 | 会话记录落库：会话 ID、模式(A/B)、发起方、目标、起止时间、时长、结果（成功/拒绝） | P0 |
| M4-2 | 操作记录落库：会话内文本日志（发起截图/鼠标点击坐标/断开等操作类型与时间） | P0 |
| M4-3 | 管理端审计列表，支持按终端、时间范围筛选 | P0 |
| M4-4 | 不做视频录像、不做回放——纯文本审计 | P0 |

### 3.5 M5 MCP Server / AI 问答

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| M5-1 | 服务端内置 MCP Server，暴露 5 个只读工具：list_endpoints / get_endpoint_detail / get_active_sessions / query_audit_log / get_screenshots | P0 |
| M5-2 | 工具从 Server HTTP API 读取数据，不直连 DB | P0 |
| M5-3 | 管理端 AI 助手页：内嵌对话框，管理员自然语言提问，AI 调用 MCP 工具实时作答 | P1 |

| 编号 | 需求描述 | 优先级 |
|------|---------|-------|
| EX-1 | 远控会话内，管理端向被控端发起文件传输（上传/下载） | P1 |

## 4. 非功能需求

| 编号 | 需求描述 |
|------|---------|
| NF-1 | **数据不出网**：所有数据在内网流转，禁止外传第三方 |
| NF-2 | **信创兼容**：支持 loongarch64/aarch64 交叉编译；X11 会话（xcap/enigo 锁 X11）|
| NF-3 | **零外部依赖部署**：SQLite 单文件，无需额外安装数据库 |
| NF-4 | **JWT 鉴权**：管理端 API + WS admin 连接需 Bearer Token；Agent 注册无需登录 |
| NF-5 | **审计降级**：DB 不可用时审计 best-effort 降级，实时远控链路不受影响 |
| NF-6 | **TLS**：服务端 + 信创端用 rustls（纯 Rust，无 openssl 依赖）；Windows 端用系统 SChannel |

---

## 5. 约束与假设

- demo 阶段：WSL 环境无真实 X11，截屏用合成占位帧（OHMYDESK_FAKE_CAPTURE=1）；真实信创物理机无此限制
- 信创真机（LoongArch）适配仅交叉编译验证，不保证 GPU 利用率等硬件细节
- demo 阶段 LLM 可用 Claude API；生产替换为内网私有化大模型

## 6. 外部接口

| 接口 | 协议 | 说明 |
|------|------|------|
| Agent ↔ Server | WebSocket（wss://host/ws） | 统一信封 `{from,to,ts,payload}` |
| Web ↔ Server | HTTP REST `/api/*` + WebSocket `/ws` | JWT 鉴权 |
| MCP ↔ Server | HTTP `/api/*` | MCP Server 读取只读数据 |
| MCP ↔ AI | stdio（MCP 协议） | Claude Desktop 或内网 AI 接入 |
