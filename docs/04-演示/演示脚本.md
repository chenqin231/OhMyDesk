## 第一段：开场白（1 分钟）

**台词参考**：

> "我们来解决一个真实的问题——企业内网里有一大批信创终端 PC，远程控制不了、管控不了、审计不了。消费级远控需要公网中转，物理隔离网根本用不了；堡垒机只管服务器，员工 PC 没人管。
>
> OhMyDesk 用 Rust 全栈，做了一个纯内网、数据不出网的终端远程安全管控平台。今天我来演示五大核心功能。"

**展示内容**：架构图（`docs/03-设计/` 中架构拓扑图）

## 第二段：五大模块演示（6 分钟）

### Demo 1 — 终端资产管理（1.5 分钟）

**操作步骤**：
1. 打开 https://rc.guoziweb.com，用 `admin / infogo@123` 登录
2. 点击左侧「终端资产」菜单
3. 展示在线终端列表，指出：
   - 绿色状态点 = 在线；灰色 = 离线
   - 信创标识图标（麒麟 OS、龙芯/鲲鹏 CPU）
   - IP、MAC、硬件摘要
4. 点击某一终端行，右侧抽屉展开硬件详情（CPU 型号/架构/内存/GPU）

**台词**：
> "Agent 启动后自动反连服务器注册。这里可以看到所有纳管终端，系统自动识别信创 OS 和 CPU 架构，无需人工录入。"

---

### Demo 2 — 一键批量截图（1 分钟）

**操作步骤**：
1. 点击「批量监控」菜单
2. 点击「一键截图」按钮
3. 等待 2-3 秒，所有在线终端截图以网格墙展示
4. 每格显示终端名称和截图时间

**台词**：
> "这是最直观的能力——一眼看全网。一个按钮，所有在线终端的当前屏幕同时出现在这里，运维值班直接用这个。"

---

### Demo 3 — 远程控制（2 分钟）

**操作步骤**：
1. 点击「远程控制」菜单
2. 从终端列表选择一台在线终端，点「远程」
3. 被控端弹出授权对话框（演示接受）
4. 管理端画面开始实时更新（截图流）
5. 演示鼠标移动 / 点击操作，画面响应
6. 点「断开」结束会话

**台词**：
> "模式 A：管理员从 Web 直接控制终端。被控端先弹窗授权，接受后画面实时呈现，鼠标键盘操作透传过去。全程走内网 WebSocket，不经过公网。"

---

### Demo 4 — 会话审计（1 分钟）

**操作步骤**：
1. 点击「会话审计」菜单
2. 展示刚才 Demo 3 产生的审计记录：
   - 连接记录：时间、发起方、目标、时长
   - 操作记录：截图请求、鼠标点击坐标、断开
3. 演示按终端/时间筛选

**台词**：
> "每次远控的连接和操作都有审计记录，文本落库，支持按终端和时间查询。不做视频录像——这里的合规核心是'谁、何时、对哪台机器、做了什么'的文本记录。"

---

### Demo 5 — AI 自然语言问答（0.5 分钟）

**操作步骤**：
1. 点击「AI 助手」菜单
2. 输入提问："现在有几台在线终端？"
3. AI 调用 MCP 工具，返回实时在线终端数
4. 再问："今天有哪些远程会话记录？"

**台词**：
> "平台内置 MCP Server，管理员用自然语言问任何管控数据，AI 实时调工具取答案。MCP Server 也在内网，数据不出网。"

---

## 第三段：总结亮点（1 分钟）

**三大差异化**：

1. **纯内网，数据不出网** — 物理隔离的信创内网可直接部署使用
2. **信创原生** — 自动识别麒麟/统信/龙芯/鲲鹏，面向国产化评估
3. **AI 时代** — 内置 MCP Server，管控数据供 AI 自然语言消费

**技术亮点**：
- Rust 全栈（客户端 + 服务端），单二进制部署，SQLite 零外部依赖
- 支持 loongarch64/aarch64 交叉编译，GitHub Actions CI 自动多架构构建

## 常见问题准备

| 问题 | 答案 |
|------|------|
| 与 RustDesk 有什么区别？ | RustDesk 需公网中转、面向个人消费；OhMyDesk 纯内网 + 信创标识 + 批量管控 + 文本审计 |
| 信创真机是否真正测试过？ | 代码支持 loongarch64/aarch64 交叉编译，demo 环境在 Linux x86 + WSL 模拟；真机验证需信创物理机 |
| 数据安全如何保证？ | 全程 TLS（rustls 纯 Rust），JWT 鉴权，审计落库，Agent 无法绕过服务端直连 |
| 能生产部署吗？ | 可以，单机部署即可（单二进制 + SQLite）；生产需设 OHMYDESK_JWT_SECRET 固定密钥 |
